# Backend xfail-ledger for galpy's existing test suite (Pillar 1).
#
# Format:  <backend> <nodeid>     (# starts a comment; blank lines ignored)
# Backends: jax | torch  (numpy runs IGNORE this file entirely.)
#
# Each line marks a test that currently FAILS under that backend as
# xfail(strict=True) when the suite is run with --backend=<backend>
# (see tests/conftest.py: pytest_collection_modifyitems). strict=True means a
# now-passing ledgered test XPASSes and FAILS the run, so this ledger can only
# SHRINK as the backend ports land -- a burndown.
#
# Nodeid-matching convention (robust to parametrization): a line matches a
# collected item if the line's nodeid equals EITHER the item's full
# parametrized nodeid ("...::test_x[ParamId]") OR the item's base nodeid with
# the "[...]" id stripped ("...::test_x"). A base-nodeid line therefore xfails
# EVERY parametrization of that test; a fully-qualified line xfails just that
# one case. The seed below uses base nodeids where every collected
# parametrization of a function failed, and full nodeids where only some did
# (e.g. test_energy_conservation_linear[<pot>...]).
#
# !!! THIS IS A SEED LEDGER -- INTENTIONALLY INCOMPLETE !!!
# It was produced by a representative, CPU-only, time-bounded slice of the
# heavy files run under BOTH backends with the C extension built:
#   test_potential.py   -k normalize/forceAsDeriv/2ndDeriv/evaluateAndDerivs/
#                          poisson/rforce/r2deriv/mass_spher/array_input/amp_mult
#   test_orbit.py       -k energy_symplec_longterm/orbit_setup/integrate_orbit/
#                          method_returntype/energy_conservation_linear
#   test_actionAngle.py -k actionAngleStaeckel_basic_{actions,freqs}[_c]
#   test_sphericaldf.py -k isotropic+anisotropic (hernquist/plummer/nfw)
# It does NOT yet cover the full suite. The first scheduled / workflow_dispatch
# CI run with regen=true runs the COMPLETE sharded suite and uploads a
# backend_xfail_new.txt artifact to complete this ledger (see build.yml).
#
# Known failure clusters captured by this seed:
#   * jax:   in-place array mutation (x[i]=...) in potential normalize/poisson/
#            mass; SCF/DiskSCF _gaussianQuadrature precompute (the SCF/DiskSCF
#            energy_conservation_linear cluster); isotropic/anisotropic
#            spherical-DF interpolation/sampling paths.
#   * torch: scalar-input rejection -- xp.sin/sqrt(<python float>) wants a
#            Tensor -- pervasive across the scalar potential/orbit/actionAngle
#            paths, plus the same df-module and 2nd-derivative clusters.
#
# Seed counts (unique nodeids), C extension built:
#   jax  : 40 total  (test_actionAngle.py=3, test_orbit.py=10, test_potential.py=10, test_sphericaldf.py=17)
#   torch: 154 total  (test_actionAngle.py=8, test_orbit.py=68, test_potential.py=15, test_sphericaldf.py=63)
